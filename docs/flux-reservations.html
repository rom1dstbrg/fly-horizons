Flux des réservations

Détail complet des deux circuits — de la demande au vol — avec tous les emails déclenchés
Circuit 1

Réservation à durée fixe

/reservation · type_resa = "standard"
1
Client Choisit un créneau et remplit le formulaire
Sur /reservation : durée (pack), date, heure, passagers, poids total, commentaire, code voucher ou code promo en option.
  • Les créneaux disponibles sont calculés par /api/reservation/slots (en tenant compte des réservations existantes + tampon 30 min).
  • Règle : minimum J-2 (48h d'avance).
  • Si un voucher est appliqué, sa durée est déduite du temps facturable → le montant baisse.
Montant > 0 €
→ Appel /api/reservation/checkout → Stripe Checkout
Statut créé : payment_pending
Montant = 0 € (voucher couvre tout)
→ Appel /api/reservation/submit
Statut créé : en_attente
2a
Stripe Paiement immédiat (si montant > 0)
Le client est redirigé vers Stripe Checkout (session expire en 30 min). Après paiement réussi → webhook checkout.session.completed → statut passe à en_attente.
✉️
→ Client
Confirmation de paiement — Fly Horizons
Récapitulatif : date, heure, durée, passagers, montant payé. Numéro de réservation.
🔔
→ Admin (info@fly-horizons.com)
[Réservation payée] Prénom Nom — date à heure
Résumé client + montant payé.
2b
Système Réservation gratuite (voucher total)
La réservation est créée directement, le voucher est marqué "used".
✉️
→ Client
Demande de réservation reçue — Fly Horizons
Confirmation de réception, récapitulatif du vol, mention du voucher utilisé. Numéro de réservation.
🔔
→ Admin
[Réservation] Prénom Nom — date à heure
Résumé complet avec tous les champs.
3
Admin Prépare et envoie la route au client Nouveau flux
Dans le drawer de la réservation (section Route), l'admin trace l'itinéraire sur la carte interactive — même outil que pour les vols sur mesure :
  • Cliquer sur la carte pour ajouter des waypoints numérotés, glisser pour déplacer, cliquer un point pour le supprimer. Bouton plein écran (⤢) disponible pour tracer plus confortablement.
  • Sauvegarder : enregistre les waypoints dans le champ final_waypoints (JSONB) — sauvegarde locale sans envoi. Les points sont rechargés à la prochaine ouverture du drawer.
  • ForeFlight : copie la route au format EBCI 50.4592N 04.4538E … EBCI en un clic.
  • Ajouter un message libre (commentaire du pilote), puis cliquer "Envoyer au client".
Même système que le vol sur mesure : crée une entrée dans route_proposals (statut pending), génère un token UUID unique, envoie l'email ci-dessous.
🗺️
→ Client
Fly Horizons — Votre itinéraire personnalisé pour le [date]
Message du pilote, liste numérotée des points de survol, bouton menant à /vol/proposition/[token].
3b
Client Consulte et répond sur /vol/proposition/[token]
Page identique à celle du vol sur mesure : message du pilote, carte Leaflet avec le tracé, liste ordonnée EBCI → waypoints → EBCI.
✅ J'accepte cet itinéraire
Statut proposal → accepted
Notification admin par email.
✏️ Demander une modification
Statut proposal → modification_requested
Le commentaire client apparaît dans le drawer → admin retrace et renvoie (retour étape 3).
🔔
→ Admin
[Route acceptée ✓] ou [Route modification demandée] Prénom Nom
Notification de la réponse client (commentaire joint si modification).
4
Admin Confirme date + heure → statut heure_confirmee
Bouton unique "Confirmer date + heure, envoyer l'email" depuis le drawer. Déclenche l'email ci-dessous.
  • Prérequis : au moins une route_proposal envoyée (ou l'ancien champ texte route renseigné pour les réservations antérieures). Bloqué sinon côté client et côté serveur.
  • Accessible depuis en_attente ou acompte_recu.
  • Le statut intermédiaire date_confirmee n'est plus un objectif direct — il reste atteignable en revenant en arrière depuis heure_confirmee ("Revenir planification").

→ Client
Votre créneau horaire est confirmé — Fly Horizons
Date + heure + durée. Lien /vol/itineraire/[token] pour les réservations avec route texte legacy.

Si météo défavorable ou autre. Génère un token de report unique.

Quand le client choisit sa nouvelle date via le lien :

→ Client

Circuit 2

Vol sur mesure

/vol-sur-mesure · type_resa = "perso"
1
Client Configure son vol sur la carte interactive
Sur /vol-sur-mesure : place des waypoints sur la carte Leaflet, choisit le style (direct ou pittoresque), sélectionne les escales avec taxes aéroportuaires, puis remplit ses coordonnées, date, heure, passagers, poids, voucher/coupon.
  • Les waypoints sont géocodés côté serveur (Nominatim) si leur nom n'est pas renseigné.
  • Le prix estimé = (prix_heure / 60) × durée − déduction voucher − code promo + taxes escales.
  • L'acompte = ~(acompte_perso_heure / 60) × durée.
2
Système Création de la réservation → /api/vol-sur-mesure/checkout
La réservation est créée avec statut en_attente. Un payment_token (UUID) est généré si l'acompte > 0.
📋
→ Client
Votre demande de vol sur mesure — Fly Horizons
Devis complet : date, heure, durée estimée, distance, style de vol, escales, détail du calcul du prix. Si acompte > 0 : lien de paiement immédiat inclus.
🔔
→ Admin
[Nouvelle demande vol sur mesure] Prénom Nom — date
Résumé + acompte + lien de paiement généré.
Note : Le lien de paiement envoyé ici est immédiat (avant proposition de route). Il est possible que le client paie dès réception du devis.
3
Admin Prépare la route dans /admin/vols
Dans le drawer, onglet Route :
  • La carte affiche la route demandée par le client (points gris, tracé gris) en référence.
  • L'admin clique "Route client" → waypoints copiés dans l'éditeur et optimisés (nearest-neighbor + 2-opt).
  • Ajustements manuels (glisser, ajouter, supprimer), inversion d'ordre possible.
  • Les escales apparaissent en overlay (code ICAO + taxe).
  • Sauvegarder enregistre dans final_waypoints ; ForeFlight copie EBCI … EBCI.
4
Admin Envoie la proposition de route au client
Depuis le drawer : bouton "Envoyer au client", avec un commentaire libre (message pilote).
Crée une entrée dans route_proposals (statut pending) avec un UUID unique.
Snapshot prix : la duree et l'acompte en vigueur dans la réservation au moment de l'envoi sont capturés dans route_proposals. Ce sont ces valeurs — pas l'estimé initial du client — qui seront utilisées dans le mail de paiement si la route est acceptée.
🗺️
→ Client
Fly Horizons — Votre itinéraire personnalisé pour le [date]
Message du pilote, liste numérotée des points de survol, bouton "Voir la carte et répondre →" menant à /vol/proposition/[token].
5
Client Consulte et répond sur /vol/proposition/[token]
Page légère fond blanc : message du pilote, carte Leaflet avec le tracé, timeline EBCI → waypoints numérotés → retour EBCI.
✅ J'accepte
Statut proposal → accepted
→ voir étape 6
✏️ Demander une modification
Statut proposal → modification_requested
Le client décrit ses souhaits → voir étape 5b
5b
Admin Reçoit la demande de modification
L'admin reçoit un email, corrige la route dans le drawer, et renvoie une nouvelle proposition (retour à l'étape 4).
🔔
→ Admin
[Route modification demandée] Prénom Nom
Commentaire du client joint.
6
Système Acceptation → envoi du lien de paiement
Déclenché automatiquement quand le client accepte, si acompte > 0 et pas encore payé.
Si le client avait déjà payé via le lien immédiat du devis initial, aucun nouvel email de paiement n'est envoyé (évite la double facturation).
Si le payment_token manque, un nouveau token est généré automatiquement.
💳
→ Client
Fly Horizons — Finalisez votre réservation
Date, durée, montant de l'acompte, bouton "Payer mon acompte →" menant à /api/vol-sur-mesure/pay/[token].
🔔
→ Admin
[Route acceptée ✓] Prénom Nom
Notification simple d'acceptation.
7
Client Paie l'acompte via Stripe
/api/vol-sur-mesure/pay/[token] → session Stripe Checkout → webhook → statut acompte_recu, payment_token effacé.
✅
→ Client
Confirmation de paiement — Vol sur mesure Fly Horizons
Récapitulatif : date, heure, durée estimée, montant payé, numéro de réservation.
🔔
→ Admin
[Vol sur mesure payé] Prénom Nom — date
Confirmation de paiement + nombre de waypoints.
8
Admin Confirmation de la date → statut date_confirmee
La route est déjà connue du client (via la proposition). Email de confirmation de date sans itinéraire.
📅
→ Client
Votre date de vol est confirmée — Fly Horizons
Date confirmée, durée.
9
Admin Confirmation de l'heure → statut heure_confirmee
🕐
→ Client
Votre créneau horaire est confirmé — Fly Horizons
Date + heure + durée estimée.
10
Admin Vol effectué → statut vol_effectue
🛬
→ Client
Merci pour votre vol — Fly Horizons
Email post-vol avec lien sondage de satisfaction /satisfaction/[id].
